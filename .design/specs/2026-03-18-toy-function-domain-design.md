# Toy Function Discovery Domain - Design Spec

**Date**: 2026-03-18
**Goal**: Create a minimal, self-contained domain for validating the auto-scientist pipeline end-to-end.

## Overview

A synthetic "function discovery" domain where the framework must discover the mathematical relationship hidden in noisy data. The ground truth is known, so we can objectively measure whether the pipeline improves over iterations.

**Hidden function**: `y = 2.5 * sin(1.5 * x) + 0.3 * x^2`
**Noise**: Gaussian, sigma = 0.5
**Dataset**: 200 points, x evenly spaced from [-5, 5]
**Theoretical best RMSE**: ~0.5 (the noise floor)

This function combines periodic and polynomial components, which is important: a simple polynomial fit will miss the oscillation, giving the Scientist a clear signal to iterate.

## What This Validates

1. **Ingestor**: Canonicalizes a CSV file (trivial but exercises the phase)
2. **Discovery**: Generates v00 script using domain config and data
3. **Analyst**: Reads results, produces structured JSON with scores
4. **Scientist**: Plans next hypothesis based on analysis
5. **Coder**: Implements the plan as a new version script
6. **Runner**: Executes experiment scripts, captures output
7. **State persistence**: Can resume if interrupted
8. **Score improvement**: RMSE should decrease across 2-3 iterations
9. **Critic/Debate**: GPT challenges the Scientist's plan before implementation

## Domain Structure

```
domains/toy_function/

    config.py          # TOY_FUNCTION_CONFIG
    prompts.py         # TOY_FUNCTION_DOMAIN_KNOWLEDGE
    seed/
        data/
            toy_function.csv    # Generated synthetic dataset
        generate_data.py        # Script that creates the CSV (reproducible)
```

### Dataset Generation (`seed/generate_data.py`)

Standalone script that generates `toy_function.csv` with columns `x` and `y`. Uses a fixed random seed (42) for reproducibility. The script is kept in the repo so anyone can regenerate the data.

```python
# Pseudocode
np.random.seed(42)
x = np.linspace(-5, 5, 200)
y_true = 2.5 * np.sin(1.5 * x) + 0.3 * x**2
y = y_true + np.random.normal(0, 0.5, size=len(x))
# Save as CSV with columns: x, y
```

### Success Criteria

Four criteria forming two tiers (basic and stretch):

| Name | Metric Key | Target | Required | Rationale |
|---|---|---|---|---|
| RMSE below 2.0 | rmse | max 2.0 | yes | Any reasonable nonlinear fit achieves this |
| RMSE below 1.0 | rmse | max 1.0 | yes | Decent fit, needs polynomial or trig terms |
| R-squared above 0.95 | r_squared | min 0.95 | yes | Good overall fit quality |
| RMSE near noise floor | rmse | max 0.6 | no | Stretch goal, requires both sin + polynomial |

### Domain Knowledge (`prompts.py`)

Intentionally minimal. We want the framework to discover the function, not be told it.

```
## Domain: Function Discovery

### Background
This is a synthetic function-discovery task. The data contains (x, y) pairs
generated by an unknown mathematical function with added Gaussian noise.

### Data
- 200 observations in a CSV with columns: x, y
- x ranges from -5 to 5
- y is a noisy measurement of the true function

### Goal
Discover the best-fitting mathematical expression for y = f(x).
The function may combine multiple mathematical forms (polynomial, trigonometric, etc.).

### Key Metrics
- RMSE: Root mean squared error (lower is better)
- R-squared: Coefficient of determination (higher is better, max 1.0)
- The data has Gaussian noise with sigma ~0.5, so RMSE below 0.5 is not achievable
```

## Expected Iteration Dynamics

**v00 (Discovery)**: Likely tries a simple approach (linear regression or low-degree polynomial). Expected RMSE ~2-3. R-squared ~0.7-0.8.

**v01**: Analyst notes the poor fit, residual patterns. Scientist hypothesizes higher-order polynomial or nonlinear terms. Coder implements. Expected RMSE ~1.0-1.5.

**v02**: Analyst notes residual oscillation in the polynomial fit. Scientist hypothesizes adding trigonometric components. Coder implements combined model. Expected RMSE ~0.5-0.7.

## Run Command

```bash
ANTHROPIC_MODEL=claude-sonnet-4-6 uv run auto-scientist run \
    --data domains/toy_function/seed/data/toy_function.csv \
    --goal "Discover the mathematical function that generated this data. The data has x and y columns. Find the best f(x) that minimizes prediction error." \
    --domain toy_function \
    --max-iterations 3 \
    --critics "openai:gpt-5.4-mini" \
    --output-dir experiments/toy_function
```

Uses Sonnet for all Claude agents (Analyst, Scientist, Coder, Discovery) via `ANTHROPIC_MODEL` env var. Uses `--critics "openai:gpt-5.4-mini"` (no reasoning) for the debate phase. No `--interactive` flag (ingestor should handle CSV without questions).

**Note**: If `claude_agent_sdk` doesn't respect `ANTHROPIC_MODEL`, we may need to add a `--model` CLI flag and thread it through to each agent. This will be verified during implementation.

## What's NOT in Scope

- Report generation (runs after the iteration loop, can test separately)
- Scheduling (`--schedule` flag)
- Synthesis (only triggers after N iterations, not relevant for 3)
- New framework code: this domain uses the existing pipeline as-is
